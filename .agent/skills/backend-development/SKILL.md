# Backend Development

## Goal
Implement backend code strictly according to the approved implementation plan
and Elysia-Light rules.

## Allowed Tools
- Edit backend source files
- Use Elysia-Light patterns
- Use Elysia-Light utilities
- Read Elysia-Light Reference
- Read Yukder V1 Reference
- Run backend build or type-check commands

## Steps
1. Read the `.agent/artifacts/plans.artifact.json` and take everything that is still open.
2. Implement code exactly as specified.
3. Use documented Elysia-Light utilities.
4. Use Elysia-Light patterns.
5. Read Yukder V1 Reference (when rebuild from Yukder V1).
6. Ensure code readability and explicit behavior.

## Forbidden Actions
- Modifying architectural decisions
- Introducing undocumented abstractions
- Fixing design issues silently
- Reading documentation to justify plan changes

## Output
- Updated backend source code
- Implementation notes (if issues arise)

## Artifacts
- `.agent/artifacts/plans.artifact.json`
